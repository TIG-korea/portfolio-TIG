Features

구현 기능

JWT + Redis 인증 흐름

일반 로그인, Refresh Token Rotation, 로그아웃 시 토큰 폐기, Google OAuth 로그인, HttpOnly Cookie 전환까지 인증 흐름을 단계적으로 구현했습니다. PR #55, #64, #73, #75, #94 기준입니다.

회원가입, 프로필, 마이페이지

User 엔티티 설계, 약관 동의, 회원가입 유효성 검증, 소셜 로그인 후 프로필 입력, 내 정보 수정, 비밀번호 변경, soft delete 탈퇴 처리를 구현했습니다.

PortOne 결제와 구독 검증

PortOne V1 API 기반 결제 검증, payment/subscription 엔티티, 구독 조회와 취소, 유료 기능 진입 전 구독 활성 여부 검증 로직을 구현했습니다.

관리자 회원/결제 관리 API

관리자 회원 목록 조회, 회원 상세 조회, 상태 변경, admin 응답 필드 보강, 관리자 결제내역 조회 API를 추가해 운영 관점의 조회 흐름을 분리했습니다.

Architecture Notes

구현하면서 고려한 기술 포인트

Mapingo 백엔드는 여러 팀원이 동시에 도메인을 개발하는 구조였기 때문에 인증 경계, DTO 응답 규칙, Swagger 문서, 외부 API 검증 흐름을 먼저 맞추는 것이 중요했습니다. 내 작업은 유저와 인증을 중심으로 결제/관리자 기능까지 확장되었습니다.

DTO와 Swagger 문서 분리

README의 프로젝트 규칙에 맞춰 Controller docs를 분리하고, Request/Response DTO 네이밍과 응답 예시를 맞췄습니다. 회원가입 문서 불일치 이슈 #59는 PR #62와 #65에서 responseCode 예시가 덮이는 문제까지 정리했습니다.

토큰 보안 흐름

초기 JWT + Redis 저장 구조에서 시작해 Refresh Token Rotation, 로그아웃 시 Redis 토큰 삭제, OAuth 1회용 코드 교환, HttpOnly Cookie 전환까지 순차적으로 개선했습니다. 토큰을 응답 body나 URL에 직접 노출하지 않는 방향으로 정리했습니다.

이슈 기반 검증

회원가입 LocalDate 검증 #36, 비밀번호 유효성 누락 #82, birthDate 검증 오류 #102처럼 재현 로그가 있는 이슈를 기준으로 입력 검증과 Swagger 예시를 수정했습니다. 수정 후에는 스웨거 캡처와 테스트 결과를 PR에 남겼습니다.

외부 API와 민감 정보 분리

Google OAuth, Redis, PortOne, FastAPI 서버 연동에 필요한 환경 변수와 application-secret.yml을 분리했습니다. 결제 검증은 imp_uid 기반 조회와 금액 검증 흐름으로 잡고, 테스트 채널 이슈는 PR #119, #122에서 조정했습니다.

Verification

PR과 이슈로 남긴 검증 기록

GitHub에서 확인한 내 PR은 인증/유저/결제/관리자 API에 집중되어 있습니다. 열린 이슈는 없고, 닫힌 이슈 중 회원가입 검증, Swagger 문서, 스키마/테스트 오류를 중심으로 재현 조건과 수정 결과를 연결했습니다.

01 / Signup Validation

회원가입 검증과 문서 불일치를 이슈 단위로 정리했습니다

회원가입 1차 구현 이후 약관 동의, 더미 약관 데이터, Swagger 응답 예시, 비밀번호 검증, LocalDate 검증 오류를 순차적으로 반영했습니다. #36, #59, #82, #102와 연결되는 흐름입니다.

  • 관련 PR #28, #33, #47, #62, #87, #99, #107
  • 검증 결과 약관 false, 중복 이메일, 비밀번호 정책, birthDate 어노테이션 오류를 Swagger와 테스트로 재확인

02 / Auth & OAuth

JWT, Redis, OAuth 흐름을 단계별로 고도화했습니다

초기 JWT 필터와 Redis Refresh Token 저장을 만든 뒤, 재발급 Rotation, 로그아웃 토큰 삭제, Google OAuth 가입/로그인, HttpOnly Cookie 저장 방식까지 보안 흐름을 단계적으로 개선했습니다.

  • 관련 PR #55, #64, #73, #75, #94
  • 검증 결과 Redis 저장값/TTL, 재발급 전후 토큰 불일치, OAuth 1회용 코드 교환, HttpOnly Cookie 동작을 캡처로 기록

03 / User, Admin, Payment

사용자 운영 기능과 결제 흐름까지 API 범위를 넓혔습니다

유저 정보 조회, 소셜 로그인 후 프로필 입력, 전화번호 중복 방지, 마이페이지 수정/탈퇴, 관리자 회원 관리, PortOne 결제/구독 검증과 관리자 결제내역 조회까지 구현했습니다.

  • 관련 PR #100, #103, #110, #116, #117, #113, #122, #127, #131, #135, #136
  • 검증 결과 단위 테스트, Swagger 응답, 상태 변경, soft delete, 결제 조회/검증 실패 조건을 PR 본문에 캡처로 기록
