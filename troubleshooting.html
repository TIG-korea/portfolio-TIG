Troubleshooting

프로젝트별로 문제를 재현하고 원인을 좁힌 기록

트러블슈팅은 단순히 오류를 고친 결과가 아니라, 어떤 흐름에서 문제가 발생했는지 재현하고 코드, 데이터, 설정, 배포 경계 중 어디가 원인인지 좁혀간 과정이라고 생각합니다. Mapingo와 TFT-gogo를 분리해 각 프로젝트에서 보여주기 좋은 사례만 정리했습니다.

Selection

선별 기준

재현 가능한 문제

막연한 개선보다 요청, 로그, 테스트, 화면 확인으로 문제가 반복 재현되는 사례를 우선했습니다.

원인 위치가 분리됨

컨트롤러, 필터, 트랜잭션, DB 스키마, 데이터 수집 로직처럼 어디에서 깨졌는지 설명 가능한 문제를 골랐습니다.

검증 결과가 남음

테스트 통과, JMeter 수치, Flyway 적용, GitHub PR/Issue 기록처럼 수정 후 확인 기준이 남아 있는 사례를 중심으로 다듬었습니다.

Mini Project

Mapingo

JWT 인증 예외, Redis 캐시 정합성, 회원가입 LocalDate 검증처럼 서비스 API에서 자주 만나는 문제를 재현 조건과 처리 경계 중심으로 정리했습니다.
